# 평행 접기 (계단식 팝업) (parallel-fold)

## 개요

척추에서부터 카드 안쪽으로 여러 단(레벨)이 계단처럼 겹겹이 솟아오르는 메커니즘. 단이 하나면 box-popup과 사실상 동일한 위상(topology)이고, 여러 단을 지정하면 폭이 점점 좁아지고 깊이가 점점 깊어지는 계단(staircase)을 만들 수 있다. 기본값은 `width=80mm, depth=30mm`으로 **단일 레벨**만 쓴다. `width`는 단의 폭(척추 방향, mm), `depth`는 척추로부터의 절단 깊이(mm)이며, 공식은 `height(α) = d·sin(α/2)`, `height_max = d`다. 다단 계단은 `levels: [{width, depth}, ...]` 배열로 지정할 수 있고 안쪽으로 갈수록 깊이는 증가(`d1 < d2 < d3`), 폭은 감소(`w1 > w2 > w3`)해야 한다. 하지만 UI 기본값은 다단을 쓰지 않으므로, 실제로 이 프로젝트 UI에서 도달 가능한 경로는 단일 레벨뿐이다(아래 "이전 작업" 참고).

## 작동 방식

### 레벨 구성과 클램프

레벨 목록이 명시적으로 주어지면 그 값들을 그대로 clamp하고, 없으면 단일 레벨(`width≈40, depth≈20` 상당의 기본값)로 만든다. 클램프 상한은 `maxDepth = 카드 높이/2 - 인쇄 안전 여백`, `maxWidth = 카드 폭 - 2·인쇄 안전 여백`로, 인쇄 안전 영역을 벗어나지 않게 한다.

레벨마다 다음을 계산한다:
- `accumulatedDepth`: 이전 레벨까지 누적된 깊이. 첫 레벨은 0.
- `cutDepthFromSpine = accumulatedDepth + depth`: 이번 레벨의 절단 깊이(척추 기준 절대값).
- 위쪽(척추 위) 절단: `upperStart = spineY - accumulatedDepth`, `upperEnd = spineY - cutDepthFromSpine`; 좌우 세로 절단선을 이 범위에만 그린다 — "이 레벨 자신의 띠 범위만" 자른다는 것이 핵심: 계단 조각이 카드에서 들려 올라갈 수 있도록 그 레벨의 좌우 모서리만 자유롭게 한다.
- 아래쪽(척추 아래)도 대칭으로 동일 계산.
- **바깥쪽(절단 깊이) 가로선 처리가 이 메커니즘의 핵심 로직**이다 — "한 선은 자르기 아니면 접기 둘 중 하나이지 둘 다일 수 없다"는 원칙 아래:
  - 마지막(가장 안쪽/가장 깊은) 레벨이면: 바깥 모서리는 완전한 자유단이므로 폭 전체를 CUT으로 긋고 산접기는 등록하지 않는다.
  - 마지막이 아니면(다음에 더 좁은 레벨이 얹힐 예정이면): 다음 레벨 폭보다 바깥쪽의 "어깨(shoulder)" 부분만 CUT으로 자르고, 다음 레벨 폭 안쪽 가운데 띠는 자르지 않는다 — 이 안 잘린 가운데 띠가 다음 레벨(i+1)이 스스로 그리는 산접기 경첩이 되기 때문에, 여기서는 그 위치에 대해 산접기를 절대 등록하지 않는다(중복 라벨링 방지).
- **안쪽(척추에 가까운 쪽) 산접기**: 레벨 0이면 척추 위치 자체가 산접기선; 레벨 i>0이면 `accumulatedDepth` 위치(=이전 레벨의 안 잘린 바깥 모서리 가운데 띠)가 산접기선.
- 라벨: 각 레벨 중앙에 "단계 N: depth mm" 텍스트.
- 척추 자체에는 골접기 표시 라인 하나가 추가된다(레벨 0 폭 기준 좌우 5mm 여유).

### 3D 조립 포즈 (열림 각도에 따른 동적 포즈, 다단 계단)

레벨 배열이 있으면 그대로, 없으면 단일 레벨로 만든다. 매 레벨의 국소 굽힘각은 동일하게 `γ = α/2`(box-popup의 단일 레벨 케이스와 동일한 `h_level = depth·sin(α/2)` 공식).

각 레벨의 DOM 구조를 재귀적으로 구성한다. **레벨 i는 레벨 i-1의 실제 DOM 자식**이라, 부모의 전체 변환을 상속받고 그 위에 자기 자신의 로컬 변환만 얹는다 — 이것이 "레벨 i는 레벨 i-1의 움직이는 표면에 붙어 있다"는 물리를 그대로 구현한 것이다.

- 레벨 0의 부모는 페이지 자체이고 경첩(hinge)은 척추; 레벨 i>0의 부모는 이전 레벨이고 경첩은 그 부모의 먼 쪽 모서리.
- **접힘 방향이 레벨마다 번갈아 뒤집힌다**: 짝수 레벨(0,2,4…)은 `+γ`("리저", 부모 표면에서 멀어지며 들림), 홀수 레벨은 `-γ`("트레드", 부모와 다시 평행하게 눕음). 이 교대가 없으면 (a) 실제 평탄 접기가 요구하는 산/골 교대를 어기고, (b) 회전 프레임이 나선형으로 누적되어 계단이 단조 상승하는 대신 수직을 지나쳐 이전 레벨 안쪽으로 말려 들어간다.
- 축 성분: `zc = side==='left' ? -cosA : cosA`, `transform = rotate3d(-sinA, 0, zc, angle°)` — v-fold/box-popup과 동일한 축 관례(sign만 좌우로 대칭).
- 총 높이 표시는 모든 레벨 depth의 합에 대해 한 번에 근사 계산한다 — 각 레벨이 독립적으로 접히는 정확한 누적치는 아니고 표시용 근사값이다.

## 이 프로젝트에서의 활용

3D 조립 포즈 프리뷰가 지원되는 3종 중 하나다. 단, UI 기본값이 단일 레벨만 지정하므로, 앱 UI를 통해 실제로 도달 가능한 3D 프리뷰는 단일 계단(=box-popup과 시각적으로 동일)뿐이다. 다단 계단 코드 경로(재귀 렌더링, 접힘 방향 교대)는 레벨 배열을 넘기는 별도 경로(예: 챗봇이 다단 파라미터를 생성하는 경우)가 아니면 실행되지 않는다.

## 이전 작업에서 배운 교훈

parallel-fold의 히스토리는 세 단계로 나뉜다.

1. **최초 자리표시자 단계**: 처음에는 화면 미리보기에 인라인으로 사각형/텍스트만 그리는 **자리표시자(placeholder)** 코드만 있었고, 실제 생성 로직은 아직 배선되지 않았다.
2. **도달 불가능(unreachable) 단계**: 실제 생성 로직은 인쇄 템플릿 생성 헬퍼와 선 스타일 헬퍼를 이미 import해서 쓰고 있었지만, **그 두 함수 자체가 그때까지 코드베이스에 존재하지 않았다** — 즉 이 생성기는 처음부터 완성된 기하 로직을 갖고 있었음에도 실행하면 즉시 에러가 나 "도달 불가능"했고, 화면은 그 대신 조용히 자리표시자 사각형만 그리고 있었다. 이후 커밋에서 누락된 두 헬퍼 함수를 추가하고 메커니즘 레지스트리를 도입해 실제 생성기가 처음으로 실행되게 만들었다.
3. **진짜 기하 버그 발견/수정 단계**: 실제로 처음 실행 가능해진 뒤에야 드러난 **진짜 기하 버그**를 고쳤다 — box-popup과 **완전히 같은 버그 클래스**: 각 레벨의 바깥 모서리가 자르기 배열과 산접기 배열 양쪽에 동일 좌표로 동시에 등록되고 있었다. 수정 내용:
   - 마지막 레벨: 바깥 모서리는 자유단이므로 전체 폭을 CUT으로만 남기고 중복된 산접기 등록을 삭제.
   - 마지막이 아닌 레벨: 다음(더 좁은) 레벨의 폭 바깥쪽 "어깨" 구간만 CUT으로 자르고, 가운데 띠는 자르지 않은 채로 남겨(다음 레벨이 스스로 그릴 산접기 경첩이 되므로) 여기서 이중으로 라벨링하지 않음.
   - 단일 레벨/3레벨 계단 양쪽에 대해 "자르기와 접기 두 배열에 동일 좌표 범위가 동시에 나타나지 않는지"를 별도 스크립트로 검증했다.
   - 같은 시기에 3D 계단 프리뷰(재귀 렌더링 + 레벨별 접힘 방향 교대)도 함께 추가됐다. 단일 레벨 parallel-fold는 모든 각도에서 box-popup과 픽셀 단위로 동일하게 보임을 확인(당연한 결과 — box-popup이 곧 단일 레벨 케이스이므로). 다단 경로는 **UI 기본값으로는 도달 불가능**해서, 검증을 위해 임시로 레지스트리를 3레벨로 고쳐 스크린샷을 찍은 뒤 커밋 전에 되돌렸다 — 즉 다단 계단 코드는 실제 운영 UI 경로로는 아직 한 번도 실사용자 눈에 노출된 적이 없다.

## 앞으로 작업 시 주의사항

- **"자르기 선과 접기 선이 동일 좌표에 중복 등록되지 않았는지" 항상 교차 검사한다.** 이 버그 패턴은 box-popup과 parallel-fold 양쪽에서 독립적으로 발생했다 — 이 계열에서 가장 흔한 버그 클래스이므로 새 팝업 메커니즘을 작성하거나 기존 걸 수정할 때 자르기/산접기/골접기 배열에 같은 좌표 쌍이 두 번 이상 들어가지 않는지 자동/수동으로 검사하는 습관이 필요하다.
- **다단(레벨 배열) 경로는 UI 기본값으로는 도달하지 않으므로, 실제로 여러 단을 사용하는 기능을 UI/챗봇에 노출하기 전에 반드시 실물(또는 최소한 3D 프리뷰) 검증을 다시 수행한다.** 과거에도 "임시로 기본값을 고쳐서 확인 후 되돌렸다"는 임시방편으로 검증했을 뿐, 회귀 테스트나 상시 검증 경로가 없다.
- **레벨 i>0의 산접기가 레벨 i-1이 "그리지 않은" 안 잘린 바깥 모서리와 정확히 일치하는지 좌표 단위로 재검증한다.** 두 레벨의 좌표 계산이 어긋나면 접었을 때 계단면 사이에 미세한 틈이나 겹침이 생길 수 있다.
- **3D 프리뷰에서 접힘 방향 교대(짝수 `+γ`, 홀수 `-γ`)를 건드릴 때는 특히 조심한다.** 이 교대를 제거하거나 실수로 뒤집으면 회전이 나선형으로 누적되어 계단이 상승하는 대신 이전 레벨 안쪽으로 말려 들어가는(retract) 시각적 버그가 재발한다.
- **총 높이 리드아웃은 근사값**이다(각 레벨이 부모 표면의 법선을 기준으로 독립적으로 기여하는 값의 단순 합) — 다단 계단의 실제 누적 3D 높이를 엄밀히 표시하려면 이 근사가 실제 좌표 합성과 얼마나 벌어지는지 검토가 필요하다.
